< 반영 사항 >

- 구글폼 설문을 통해 실제 사용자 요구를 수집하고, 이를 기반으로 기능적 요구사항 8개, 비기능적 요구사항 5개를 도출함
- 설문 결과에서 도출된 주요 피드백(검색 기능, 세세한 진도율, 날짜 기록, 가나다순 정렬, 간결한 UI)을 요구사항 명세에 직접 반영하여 사용자 중심의 기능 범위를 확정함
- 기능적 요구사항은 탭 단위(독서 기록 / 서재 / 통계)로 분류하고, 각 기능의 우선순위(높음 / 중간)를 지정하여 개발 순서의 근거로 활용함.
- 비기능적 요구사항으로 성능(1초 이내 반영), 신뢰성(localStorage 지속성), 호환성(크롬/엣지 최신 버전)을 명세화하여 테스트 항목 구성의 기준으로 삼음.
- 유스케이스 명세를 통해 사용자의 행동 흐름과 예외 처리 시나리오를 정의하고, 이를 개발 및 테스트 단계와 연결함.
